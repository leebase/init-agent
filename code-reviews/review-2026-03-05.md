# Code Review — 2026-03-05

## Architecture Summary

`init-agent` is a single-binary Zig CLI tool (no runtime dependencies) that scaffolds AI-agent project kits. Templates are embedded at compile time via `@embedFile` from `src/templates/`. Template variable substitution (`{{PROJECT_NAME}}` etc.) runs at scaffold time. Three profiles exist (python, web-app, zig-cli) each defining a list of `TemplateFile` entries and a directory list. `--update` iterates the same file list and overwrites contract files in place. Project detection for `--update` combines `context.md` parsing and signature file heuristics. The main risk areas are: mutable global state for batch overwrite mode, hardcoded date, subprocess output leaks, and the dual-template-tree sync requirement that has no automated enforcement.
